
## 1) Scaricare una repository GitHub in locale (clone)

### Clone via HTTPS

```bash
git clone https://github.com/USERNAME/NOME-REPO.git
```

### Clone via SSH (consigliato se usi chiavi)

```bash
git clone git@github.com:USERNAME/NOME-REPO.git
```

### Clone in una cartella specifica

```bash
git clone https://github.com/USERNAME/NOME-REPO.git percorso/cartella
```

---

## 2) Collegare una repo GitHub a un progetto locale già esistente

```bash
git init
git branch -M main
git remote add origin https://github.com/alepnt/AgenteAI
```

Verifica:

```bash
git remote -v
```

---

## 3) Caricare (push) il progetto locale su GitHub

### Primo push (obbligatorio)

```bash
git add .
git commit -m "1.0.0.0.app.0.0.0.2"
git push -u origin main
```

### Push successivi

```bash
git add .
git commit -m "Messaggio commit"
git push
```

---

## 4) Scaricare aggiornamenti da GitHub (pull)

```bash
git pull origin main
```

Oppure (se tracking già impostato):

```bash
git pull
```

---

## 5) Aggiornare repo locale senza conflitti (best practice)

```bash
git pull --rebase
```

---

## 6) Script completo “one-shot” (clone + lavoro + push)

```bash
git clone https://github.com/USERNAME/NOME-REPO.git
cd NOME-REPO
# lavori...
git add .
git commit -m "Update"
git push
```

---

## 7) Script PowerShell (Windows) automatico

```powershell
git add .
git commit -m "Auto commit"
git push origin main
```

---

## 8) Errori comuni e fix rapidi

### Repo vuota su GitHub

```bash
git push -u origin main
```

### Branch main vs master

```bash
git branch -M main
```

### Forzare push (attenzione)

```bash
git push --force
```
